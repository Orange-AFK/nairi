# Nairi

## Product Positioning

### Core Positioning

Nairi is an agent-first CMS for publishing project experience. It turns AI-agent collaboration outputs into structured, reviewable, editable, and publishable content.

## Primary Workflow

### Agent-Assisted Article Publishing

1. A project reaches a valuable milestone.
2. The user asks an agent to write a blog post about the design and development process.
3. The agent gathers permitted context and creates a draft through API-backed tools.
4. Nairi stores the draft as a CMS post with metadata and audit history.
5. The admin console shows the draft, risk checks, revision history, and publication options.
6. A human administrator approves, edits, schedules, or rejects publication.

## Human CMS Experience

### Required Admin Capabilities

1. Content creation and editing.
2. Draft review and publication control.
3. Media management.
4. MDX component governance.
5. Agent task review.
6. API token and scope management.
7. Audit log inspection.
8. Site settings and SEO management.

## Agent Experience

### Agent Responsibilities

1. Generate structured drafts.
2. Explain reasoning and source context.
3. Warn about risky content or MDX usage.
4. Request elevated capability when required.
5. Respect API scopes and human review rules.

## Product Differentiation

### Why Nairi Exists

Traditional blogs require manual transformation from project work to article. Nairi makes that transformation API-governed and agent-assisted, while preserving human control.
